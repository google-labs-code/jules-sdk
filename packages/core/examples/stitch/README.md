# Stitch Integration Example

This example demonstrates how to use the Jules SDK to integrate a website design generated by Stitch into an existing React codebase.

It uses the `@google/stitch-sdk` to programmatically generate a new design, retrieves the HTML download URL, and uses a Jules session to analyze the provided link, extract the code/design components, and generate a pull request applying the new design.

## Prerequisites

1.  **Node.js** (v18+)
2.  **JULES_API_KEY**: Ensure this environment variable is set.
3.  **STITCH_API_KEY**: Ensure this environment variable is set to use the Stitch SDK.

```bash
export JULES_API_KEY="your-jules-api-key"
export STITCH_API_KEY="your-stitch-api-key"
```

## Running the Example

Navigate to this directory and run:

```bash
npm start
```

## How It Works

1.  **Stitch Generation**: The example creates a new Stitch project and generates a design (e.g., a dark theme hero section) using the Stitch SDK.
2.  **Configuration**: The example initializes a Jules session with a `source` set to a target repository and branch (mocked in the script, you should adjust it to your own test repository).
3.  **Prompt**: It provides a prompt containing the HTML download URL of the generated Stitch design and instructions to implement the design into the specific pages/components of the repository.
4.  **Execution**: Jules will launch an agent that explores the repository, reads the Stitch design, plans the implementation, modifies the source code, and creates a pull request.
5.  **Monitoring**: The script uses `.stream()` to monitor the progress of the agent.

## Notes

- Be sure to update `GITHUB_REPO` and `BASE_BRANCH` in `index.ts` to a repository you have access to if you want to see a real PR generated.
- This is a demonstration script. In a real-world scenario, you might trigger this script via a webhook from Stitch or another CI/CD process.
