# Stitch Integration Example

This example demonstrates how to use the Jules SDK to integrate a website design generated by Stitch into an existing React codebase.

It assumes you have a mock or real URL to a Stitch project, and uses a Jules session to analyze the provided link, extract the code/design components, and generate a pull request applying the new design.

## Prerequisites

1.  **Node.js** (v18+)
2.  **JULES_API_KEY**: Ensure this environment variable is set.

```bash
export JULES_API_KEY="your-jules-api-key"
```

## Running the Example

Navigate to this directory and run:

```bash
npm start
```

## How It Works

1.  **Configuration**: The example initializes a Jules session with a `source` set to a target repository and branch (mocked in the script, you should adjust it to your own test repository).
2.  **Prompt**: It provides a prompt containing the URL to the Stitch design and instructions to implement the design into the specific pages/components of the repository.
3.  **Execution**: Jules will launch an agent that explores the repository, reads the Stitch instructions/design, plans the implementation, modifies the source code, and creates a pull request.
4.  **Monitoring**: The script uses `.stream()` to monitor the progress of the agent.

## Notes

- Be sure to update `GITHUB_REPO` and `BASE_BRANCH` in `index.ts` to a repository you have access to if you want to see a real PR generated.
- This is a demonstration script. In a real-world scenario, you might trigger this script via a webhook from Stitch or another CI/CD process.
